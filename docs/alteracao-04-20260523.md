# Resumo da Alteração
**Data:** 23/05/2026
**Responsável:** JoyBoy (Agent IA)
**Objetivo:** Implementação de Logs, Swagger (Documentação) e Testes (Jest)

## O Quê
A aplicação agora conta com observabilidade (Logs), uma vitrine para a API (Swagger) e um ecossistema de testes robusto (Jest + Supertest).

## Por Quê
Embora a base de código já estivesse padronizada e fortemente tipada em Clean Architecture, nós precisávamos de:
1. **Logs:** Para facilitar o debug em produção (através do arquivo `logs/error.log` e do nosso LogRepository nativo).
2. **Swagger:** Para estabelecer um contrato fixo e visual entre o Backend e o Frontend, tornando a manutenção fácil.
3. **Jest:** Para ter um ciclo de desenvolvimento seguro contra regressões. Com os testes unitários e mocks, agora conseguimos testar as regras de negócio de `UserServiceImpl` isoladas do banco de dados, e com o Supertest garantimos os fluxos de ponta a ponta (como o Login HTTP).

## Como (Resumo das Modificações)
1. **Logs:**
   - Instalação de `morgan` (para os acessos HTTP) e `winston` (para gerenciamento de logs em arquivo).
   - Criação do `src/utils/logger.ts` e do `src/middlewares/errorMiddleware.ts`.
   - Adição do middleware de erros globais no `src/app.ts`, gravando os erros fatais diretamente no LogRepository.
2. **Swagger:**
   - Instalação de `swagger-ui-express` e `yamljs`.
   - Criação de `docs/swagger.yaml` documentando `/auth/login` e `/admin/users` (podendo escalar).
   - Rota visual habilitada em `/api-docs`.
3. **Testes (Jest):**
   - Instalação e configuração de `jest`, `ts-jest` e `supertest`. Ajuste no `tsconfig.json`.
   - Criação de Mocks In-Memory (ex: `MockUserRepository`) para simular o banco de dados.
   - Implementação de Teste Unitário completo para o `UserServiceImpl.create` (`UserServiceImpl.spec.ts`), validando falhas por e-mail inválido e privilégios ausentes.
   - Implementação de Teste de Integração para a rota `/auth/login` (`Login.spec.ts`), validando o sucesso e falha via HTTP e Mocks de Injeção do Repository e bcrypt.
