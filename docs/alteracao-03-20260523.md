# Resumo da Alteração: Tipagem de Promises nos Repositórios (Screaming Architecture)

- **Data**: 23/05/2026
- **Autor**: JoyBoy (AI Agent)
- **Escopo**: Refatoração da Camada de Repositórios e Serviços (Tipagem Estrita)

## O Quê
Refatoramos todas as Interfaces de Repositório (`src/repositories/`) para substituir os genéricos `Promise<any>` por Tipos de Domínio Estritos (`Promise<User>`, `Promise<Contract>`, `Promise<Profile>`, etc.) com base na pasta `src/models/database/`. Adicionalmente, mapeamos todos os retornos do ORM Sequelize usando `.get({ plain: true })` para garantir o isolamento da regra de negócio.

## Por Que
Na fase anterior de refatoração, a extração das interfaces ainda permitia a passagem de tipos `any`, o que enfraquecia o `TypeScript` e abria margem para erros silenciosos nos Usecases. A tipagem estrita no contrato força os Serviços a consumirem apenas os dados previstos no Domínio puro, bloqueando métodos poluentes do ORM (ex: `.save()`, `.destroy()`) vazarem para a regra de negócio.

## Como foi feito
1. **Modelos de Domínio**: Adicionamos propriedades vitais que faltavam ao modelo `Profile.ts` (ex: `tax_id`, `phone`, `street`, etc.). Criamos o arquivo `ContractItem.ts`.
2. **Interfaces**: Atualizamos `BillingRepository.ts`, `ClientRepository.ts`, `ContractRepository.ts`, `ProfileRepository.ts`, `ServiceRepository.ts`, e `ClientDashboardRepository.ts` para devolverem os tipos literais.
3. **Mapeamento**: Adicionamos `return item.get({ plain: true })` no momento do retorno nas classes `SequelizeXXXRepository` dentro da pasta `database/`.
4. **Resolução de Erros**:
   - Ajustamos o `ClientDashboardServiceImpl` que chamava `profile.createdAt` em vez do padrão de banco de dados `created_at`.
   - Modificamos a assinatura do contrato e parcelas no `BillingServiceImpl` e `ContractServiceImpl` (`status as 'pending' | 'paid'`).
   - Corrigimos o `UserServiceImpl` que tentava chamar `.update` no Profile ao invés de `.upsert`.
5. **Validação Final**: Toda a base foi submetida a `tsc --noEmit` repetidas vezes até alcançarmos 0 erros e compilação limpa. O arquivo temporário `plano.md` foi deletado após a conclusão.
