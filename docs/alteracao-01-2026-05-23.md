## 📝 Resumo (O quê, por que e como)
- **O quê**: A base de código completa (exceto dependências, `.git` e builds locais) foi replicada via `rsync` para a pasta `/home/cloves/Documentos/Github/back-arcadian`. As dependências foram reinstaladas, e as configurações base (como o acesso ao Supabase) foram migradas com segurança.
- **Por que**: Evitar conflitos e quebra do sistema em produção (o backend original atende a aplicação legada). Com essa migração, as novas rotas, o novo frontend e as novas tabelas de BD podem ser trabalhadas em um ambiente de desenvolvimento isolado.
- **Como**: Utilizando comando terminal de transferência segura (`rsync`) e instalando os pacotes. Validadas as configurações de CORS para confirmar compatibilidade de integração sem atritos.
