# Resumo da Alteração
**Data:** 22/05/2026
**Responsável:** JoyBoy (Agent IA)
**Objetivo:** Refatoração Completa da Arquitetura (Screaming Architecture & Clean Code)

## O Quê
Toda a base de código do backend foi reestruturada para seguir uma arquitetura fortemente orientada a domínios (Screaming Architecture), eliminando pastas genéricas (como `services` e `libs`) e agrupando as funcionalidades por casos de uso (`usecases`).

## Por Quê
A aplicação antiga possuía alto acoplamento, com lógicas de negócio misturadas dentro de `Controllers` e `Routers`. Além disso, o acesso a dados estava pulverizado. A nova estrutura isola responsabilidades em 3 camadas:
1. **Controller:** Lida puramente com Request/Response HTTP.
2. **Service (Interface e Impl):** Contém 100% da regra de negócio.
3. **Repository:** Único ponto de contato com o banco de dados (Sequelize).

Isso garante um código escalável, testável e de fácil manutenção para o futuro do Arcadian Hub.

## Como (Resumo das Modificações)
1. **Reestruturação Raiz (Fase 1):** 
   - Criação de pastas `config`, `utils`, `repositories`.
   - Implementação do `SequelizeSingletonConnection`.
2. **Domínios (Fases 2 e 3):**
   - Refatoração total dos módulos: `auth` (login, password, verify-token), `admin` (billing, clients, contracts, logs, service, stats, users) e `client` (dashboard, profile).
   - Eliminação de rotas genéricas como `/users/` e `/clients/` e realocação para `admin/users/` ou `client/profile/`, centralizando a orquestração via *Controllers* em `src/routers/`.
3. **Rotas (Fase 4):**
   - Remoção completa de lógicas do Express Router (ex: limpeza do `ClientRouter.ts`). 
   - Mapeamento puro por `controller.action.bind(controller)`.
4. **Limpeza:**
   - Exclusão de todo o código morto.
   - Tipagem rigorosa (Exit Code `0` no `npx tsc --noEmit`).
